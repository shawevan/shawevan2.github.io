发表于 2021-03-17

AT89S51单片机的硬件组成


AT89S51内部各部件通过内部单一的总线连接而成,其基本结构依旧是CPU加外围部件的传统微型计算机结构模式,但CPU对各种外围部件的控制方式是采用 特殊功能寄存器 (Special Function Resister,SFR)进行集中控制.

图中各部件介绍

  1. CPU: 8位CPU
  2. 数据存储器(RAM):内部128B,可扩充64KB
  3. 程序存储器(Flash存储器):用来存储烧写的代码程序,AT89S51内部有4KBFlash存储器,可扩充64KB
  4. 中断系统:具有五个中断源,2级中断优先级
  5. 定时/计数器:内部有两个16位定时/计数器,具有4种工作方式
  6. 串行口:一个全双工异步串行口(UART),具有4种工作方式.可进行串行通信,扩展并行I/O口,还可以与多个单片机相连构成多机串行通信系统
  7. 8位并行口:P0、P1、P2、P3
  8. 特殊功能寄存器:特殊功能寄存器:AT89S51单片机定义了26个特殊功能寄存器,用来控制、监视和管理各功能部件,通过增加SFR可以扩展增加功能部件,特殊功能寄存器在内存中占用一个字节,分布在RAM区的80H~FFH地址范围中.单片机初始化时,未定义的引脚在调用时只能使用该引脚对应的地址,如存储P1.0~P1.8引脚状态的内存地址是0x90,使用起来不方便,可以使用 sfr 将地址赋值给变量P1,即 sfr P1 = 0x90,之后需要调用引脚就可以使用P1变量而不必使用0x90地址.
  9. 看门狗定时器(WDT):当单片机由于干扰而是程序陷入死循环或跑飞状态时,看门狗定时器可引起单片机复位,使程序恢复正常运行

引脚功能

目前AT89S51多采用40引脚塑料双列直插封装(DIP),与各种8051单片机引脚兼容,如图

40个引脚按 功能可以分为三类
(1) 电源及时钟引脚:电源(Vcc、Vss)、时钟(XTAL1、XTAL2)
(2) 控制引脚: $\overline{\text{PSEN}}$、ALE/$\overline{\text{PROG}}$、$\overline{\text{EA}}$/Vpp、RST
(3) 并行I/O口引脚:P0、P1、P2、P3,为4个8位并行I/O口的外部引脚

计算机网络1:概论

发表于 2021-03-15 更新于 2021-03-19

计算机网络

21世纪的重要特征就是 数字化、网络化、信息化 ，是一个以 网络 为核心的 信息时代 。

当今我们比较熟悉的网络有三大类:

  • 电信网络
  • 有线电视网络
  • 计算机网络

电信网络主要提供电话、电报以及传真服务，有线电视网络向用户提供电视节目，计算机网络使用户能在计算机之间传输数据文件。

计算机网络是其中 发展的最快 并且起到 核心作用 的网络。

由于计算机网络能够承担电信网络以及有线电视网络的功能,所以有” 三网融合 “的观点,即将三种网络融合成一种网络以提供三种网络的服务。

互联网

互联网,即 Internet ,也翻译作因特网,但译作互联网能够体现出Internet的特点,即 互联网是由数量极大的各种计算机网络互连起来的 ,采用互联网这个译名能够体现出Internet最主要的特征。

互联网具有两个重要的基本特点，连通性和共享

  1. 连通性: 即使用户之间物理距离很远，也可以较为经济便捷地交换各种信息。
  2. 资源共享: 这是计算机网络与其他网络的本质区别。

互联网概述

网络的网络

计算机网络(简称 网络 )由若干 节点(任何计算机、智能设备)以及连接这些节点的 链路 组成，网络中的节点可以是计算机、集线器、交换机或路由器等。

网络之间还可以使用路由器连接在一起，组成一个更大的网络,这样的网络称为互连网,因此互连网是”网络的网络”
习惯上,把与网络连接的计算机称为 主机(host)

互连网（internet，小写i）是一个通用名词，它泛指有多个计算机网络互连而成的计算机网络，这些网络之间的通信规则可以任意选择，不一定非要用TCP/IP协议。

互联网（Internet，大写I）是一个专用名词，也就是因特网。专指当前全球最大的、开放的、由众多网络相互连接而成的特定互连网，它采用TCP/IP协议族作为通信的规则

网络把许多计算机连接在一起,而互连网把网络通过路由器连接在一起

互联网基础结构发展的三个阶段

第二阶段的特点是建成了 三级结构的互联网,分为 主干网、地区网和校园网

第三阶段的特点是逐渐形成多层次 ISP 结构的互联网,ISP(Internet Service Provider,网络服务提供商),根据服务范围的大小以及IP地址数目大小分为三个层次,主干ISP、地区ISP和本地ISP

在这样的架构下,主机A要与不同地区的主机B连接需要以下步骤:
主机A $\rightarrow$ 本地ISP $\rightarrow$ 地区ISP $\rightarrow$ 主干ISP $\rightarrow$ 地区ISP $\rightarrow$ 本地ISP $\rightarrow$ 主机B
随着网上数据流量的增长,为了更快地转发分组,更有效地利用网络资源,就产生了互联网交换点(IXP,Internet eXchange Point)
IXP允许两个网络直接连接并交换分组,而不再需要第三个网络来转发分组

由欧洲原子核研究组织开发的 万维网WWW (World Wide Web)是互联网在应用层的功能之一,被广泛使用在互联网上

互联网的组成

互联网从工作方式上看可以分为两大块:
(1) 边缘部分 由所有连接在互联网上的主机组成
(2) 核心部分 由大量网络以及连接这些网络的路由器组成,除路由器外不包含计算机

端系统通信

主机A与主机B的通信,实质上是运行在主机A上的一个程序和运行在主机B上的一个应用程序的通信,简称为 “计算机之间通信”

端系统通信的两种方式
1.客户-服务器方式(C/S)

客户(client) 和 服务器(server) 都是指通信中涉及的两个应用进程
客户是服务请求方,服务器是服务提供方
服务请求方和服务提供方都要使用网络核心部分所提供的服务

主要特点:
客户程序:

  1. 被用户调用后运行,主动向服务器发起通信, 必须知道服务器程序的地址
    2.不需要特殊的硬件和复杂的操作系统

服务器程序:

  1. 是一种专门用来提供某种服务的程序, 可同时处理多个远地或本地客户请求
  2. 不断运行, 被动等待并接受来自各地用户的通信请求
  3. 一般需要强大的硬件和复杂的操作系统支持

客户与服务器通信关系建立后,通信是双向的,客户和服务器都可以收发数据

对等连接方式

对等连接方式(peer-to-peer,简称P2P)并不区分客户与服务器,双方都可以下载对方存储的文件,可支持上百万个用户同时工作,其应用由面对面快传、视频电话等

事实上,P2P从本质上依旧是C/S方式,只是同一台主机既是客户又是服务器

互联网核心部分

在网络核心部分起特殊作用的是 路由器(router),它是一种专用计算机(但不叫作主机,因为其一般情况下没有有信息价值的数据),路由器是实现 分组交换(packet switching)的关键构件,其任务是 转发收到的分组 ,这是网络核心部分最重要的功能

为了弄清分组交换需要先理解电路交换

电路交换

电路交换伴随电话线路而诞生,交换(switching)就是按照某种方式动态地分配传输线路的资源
拨打电话时,拨打与接听方间需要建立一条专用的物理通路,旁人无法插入,直到通话结束将线路资源释放,这种必须通过
建立连接(占用通信资源) $\rightarrow$ 通话(持续占用通信资源) $\rightarrow$ 释放连接(归还通信资源)
三个步骤的交换方式称为 电路交换
电路交换具有独占性的特征,它必定是面向连接的,这表示它有具体的物理线路

Python项目1:彩色图片转灰度图

发表于 2021-02-21

源码

保存为ascii.py


包括对两个模块PIL与argparse的使用

实验原理

通过特定的公式将 RGB值 转化为对应的 灰度值 ，将要用的字符与灰度值对应
公式: 

gray = 0.2126 * r + 0.7152 * g + 0.0722 * b

(_这个公式并不是一个真实的算法,而是简化的sRGB IEC61966-2.1公式,真实的公式更复杂一些_)

代码解析

argparse模块

　　argparse是python的 命令行解析的标准模块 ,内置于python。这个库可以 直接在命令行中就可以向程序传入参数并让程序运行 。

使用argparse处理命令行参数,获取输入的图片路径、输出字符画的宽和高以及输出路径。

get_char是RGB转字符的函数 

if alpha == 0:  
    return ' '

判断像素点是否为白色,是则输出一个空格
因为字符集只有70个,无法与0~255一一对应,于是这里将256个灰度值分为多个unit

unit = (256.0 + 1)/length

所以返回的灰度值对应的字符下标为:int(gray/unit)

最后一个步骤,对图片进行处理
这个步骤的代码放到if __name__ == '__main__'中,这样当源码被import时,这部分的代码块不会执行
图片处理的步骤如下:

  1. 使用PIL的Image.open打开图片,保存到im中
  2. 使用im.resize()调整图片大小,对应输出的字符画宽度和高度,Image.NEAREST表示输出低质量图片
  3. 遍历每行的像素点获取其RGB值,调用get_char()函数转成对应字符串
  4. 将所有的像素对应的字符拼接在一起成为一个字符串txt
  5. 打印输出字符串txt
  6. 如果执行时配置了输出文件,将打开文件将txt输出到文件,如果没有,则默认输出到output.txt文件

其中是通过PIL模块的getpixel获取像素点的RGB值 

get_char(*im.getpixel((j,i)))

im.getpixel((j,i))可以获取(j,i)像素点的RGB值(有时会包含alpha值),返回一个 元组 , 使用 * 可以将元组作为参数传递给get_char(),同时元组中的每个元素都对应到get_char()函数的每个参数。

转义

Markdown使用了很多特殊符号来表示特定的意义,如果需要显示特定的符号则需要使用转义字符,Markdown使用反斜杠转义特殊字符: 

**文本加粗**
\*\* 正常显示星号 \*\*

输出结果为:

文本加粗
**正常显示星号**

Markdown支持以下这些符号前面加上反斜杠来帮助插入普通符号: 

\   反斜线
`   反引号
*   星号
_   下划线
{}  花括号
[]  方括号
()  小括号
#   井字号
+   加号
-   减号
.   英文句点
!   感叹号
